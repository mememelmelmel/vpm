# Mememelmelmel VPM

VPM (VRChat Package Manager) listing for [めめめるめるめる](https://mememelmelmel.booth.pm/) packages.

## Install via VCC

One-click (opens VCC and adds this listing):

```
vcc://vpm/addRepo?url=https%3A%2F%2Fmememelmelmel.github.io%2Fvpm%2Findex.json
```

Or add this URL manually in VCC → Settings → Packages → Add Repository:

```
https://mememelmelmel.github.io/vpm/index.json
```

After adding the listing, open your project in VCC → Manage Packages and enable
**Mel Hair Profiles**. Its dependency **Mel Profile Applier** is resolved
automatically from this same listing.

## Included packages

| Package | Source |
|---|---|
| `mememelmelmel.hair-profiles` | https://github.com/mememelmelmel/hair-profiles |
| `mememelmelmel.profile-applier` | https://github.com/mememelmelmel/profile-applier |

## How it works

`index.json` is generated by merging the per-package listings declared in
[`sources.json`](sources.json). Each source is the published `index.json` of a
single-package listing, which remains the source of truth for its own versions.
This repo only aggregates them into one listing so a single VCC repository
exposes every package.

To add another package, append its published listing URL to the `sources`
array in `sources.json`.

### Rebuild

```
node scripts/build-listing.mjs
```

The [Build VPM Listing](.github/workflows/build-listing.yml) workflow runs this
daily (and on demand) and commits any changes. GitHub Pages serves `index.json`
from the repo root.
